## 软件介绍与安装

### 软件概述

**NovaLocal AI Novelist** 是一款专为小说创作者设计的桌面应用程序，它集成了 Google Gemini AI 模型，提供从灵感生成到完整小说创作的全流程辅助工具。

#### 主要功能特点：
- **AI 辅助创作**：从灵感、角色、大纲到章节的完整创作流程
- **本地数据存储**：所有项目数据保存在本地，保护隐私
- **知识库管理**：上传参考资料并在创作中引用
- **多模型支持**：可配置不同的 AI 模型参数
- **流式输出**：实时查看 AI 生成内容
- **历史记录**：完整记录所有 AI 生成内容

#### 适用人群：
- 小说作家和写作爱好者
- 需要 AI 辅助的创作者
- 希望提高写作效率的作者
- 学习小说创作的新手

### 安装与配置

#### 系统要求
- **操作系统**：Windows 10/11（也支持 macOS 和 Linux）
- **Node.js**：18+ 版本
- **内存**：建议 8GB 以上
- **存储空间**：至少 500MB 可用空间

#### 安装步骤

2. 双击运行 `NovaLocal AI Novelist.exe`

#### 首次启动设置

1. **AI 模型配置**（必需）：
   - 点击右上角的模型名称（如"未选模型"）
   - 进入"模型设置"
   - 配置 Google Gemini API 密钥
   - 选择或创建模型配置

2. **创建第一个项目**：
   - 在灵感页面输入初始想法
   - 点击"生成书名与简介"
   - 开始你的创作之旅

## 核心功能使用指南

### 项目管理

#### 创建新项目
1. 启动应用后，自动进入"灵感生成"页面
2. 在文本框中输入你的初始灵感
3. 点击"生成书名与简介"
4. 系统会自动创建新项目并命名

#### 保存和加载项目
- **自动保存**：所有更改都会自动保存到本地
- **手动保存**：无需手动操作，系统自动处理
- **加载项目**：重新启动应用后，上次的项目会自动加载

#### 项目导入/导出
1. **导出项目**：
   - 点击灵感页面右上角的"备份"按钮
   - 选择保存位置
   - 项目数据将导出为 JSON 文件

2. **导入项目**：
   - 点击灵感页面右上角的"导入"按钮
   - 选择之前导出的 JSON 文件
   - 确认导入，项目数据将恢复

#### 项目备份与恢复
- 数据存储位置：`%APPDATA%/novalocal-ai-novelist/`
- 定期备份建议：使用导出功能备份重要项目
- 恢复出厂设置：在设置中可进行数据重置

### 创作流程详解

NovaLocal AI Novelist 采用六步创作流程，引导你完成完整的小说创作：

#### 步骤0：灵感生成

**功能说明**：将零散的想法转化为完整的小说创意

**操作步骤**：
1. **输入灵感碎片**：
   - 在文本框中输入任何想法：题材、关键词、背景设定等
   - 示例："一个在魔法学院学习的普通学生，突然发现自己有特殊能力"

2. **引用知识库**（可选）：
   - 点击"上传新资料"上传参考文档
   - 选择要引用的知识条目
   - 支持的文件格式：.txt, .md, .json, .csv

3. **选择输出模式**：
   - **流式输出**：实时查看生成过程（推荐）
   - **传统输出**：一次性显示完整结果

4. **生成内容**：
   - 点击"生成书名与简介"
   - 等待 AI 生成结果
   - 可随时暂停、继续或停止生成

5. **编辑结果**：
   - 修改生成的书名
   - 编辑 AI 生成的简介
   - 保存到项目

**实用技巧**：
- 灵感越具体，生成结果越精准
- 上传相关参考资料可提高生成质量
- 使用流式输出可实时调整生成方向

#### 步骤1：知识库管理

**功能说明**：管理创作所需的所有参考资料

**操作步骤**：
1. **上传资料**：
   - 点击"上传新资料"按钮
   - 选择文本文件
   - 系统自动提取内容并分类

2. **组织知识**：
   - 查看所有上传的资料
   - 按类别筛选（灵感、角色、大纲等）
   - 编辑知识条目名称和内容

3. **引用知识**：
   - 在后续步骤中可选择引用特定知识
   - AI 会基于引用的资料进行生成

**实用技巧**：
- 上传角色设定、世界观文档等参考资料
- 为知识条目添加清晰的名称便于查找
- 定期整理知识库，删除不需要的资料

#### 步骤2：角色构建

**功能说明**：创建小说中的角色体系

**操作步骤**：
1. **创建角色**：
   - 点击"添加角色"按钮
   - 填写角色基本信息：姓名、年龄、性别等
   - 添加角色背景和性格特点

2. **AI 生成角色**：
   - 使用 AI 自动生成角色背景故事
   - 基于灵感生成相关角色
   - 批量生成多个角色

3. **角色关系**：
   - 设定角色之间的关系
   - 可视化角色关系图
   - 调整角色重要性

**实用技巧**：
- 为主角创建详细档案
- 为配角设定关键特征即可
- 使用关系图理清角色网络

#### 步骤3：大纲设计

**功能说明**：创建小说的整体结构

**操作步骤**：
1. **生成大纲**：
   - 基于灵感和角色生成小说大纲
   - 选择大纲模板或自定义
   - 调整故事结构和节奏

2. **编辑大纲**：
   - 自由编辑 AI 生成的大纲
   - 调整章节划分
   - 设定关键情节点

3. **细化结构**：
   - 为每个部分添加详细说明
   - 设定冲突和发展
   - 确保逻辑连贯性

**实用技巧**：
- 先生成整体框架再细化
- 确保每个章节有明确的目的
- 保持故事节奏的平衡

#### 步骤4：章节规划

**功能说明**：为每个章节创建详细规划

**操作步骤**：
1. **生成章节**：
   - 基于大纲自动生成章节列表
   - 为每个章节生成概要
   - 设定章节要点和关键事件

2. **调整顺序**：
   - 拖拽调整章节顺序
   - 合并或拆分章节
   - 添加过渡章节

3. **进入写作**：
   - 点击章节进入写作编辑器
   - 开始撰写正文内容

**实用技巧**：
- 为重要章节添加详细要点
- 保持章节长度相对均衡
   - 考虑读者的阅读节奏

#### 步骤5：正文创作

**功能说明**：实际的写作编辑环境

**操作步骤**：
1. **写作界面**：
   - 全屏写作编辑器
   - 左侧章节导航
   - 右侧写作区域

2. **AI 辅助写作**：
   - 点击"AI 续写"继续当前段落
   - 使用"润色优化"改进文笔
   - "扩写细节"丰富场景描写

3. **内容管理**：
   - 自动保存所有更改
   - 查看章节历史版本
   - 导出章节内容

**实用技巧**：
- 先完成初稿再优化
- 合理使用 AI 辅助，保持个人风格
- 定期保存和备份

## 高级功能使用

### 全局助手

**功能**：随时获取 AI 写作建议

**使用方法**：
1. 点击右下角的浮动助手按钮
2. 输入你的问题或需求
3. 获取即时 AI 建议
4. 可将建议插入到编辑器中

**适用场景**：
- 写作卡顿时获取灵感
- 需要特定描写建议
- 角色对话优化
- 情节发展建议

### 模型设置

**功能**：自定义 AI 模型行为

**配置选项**：
1. **API 密钥管理**：
   - 添加/更换 Google Gemini API 密钥
   - 测试连接状态

2. **模型参数**：
   - **温度**：控制创造性（0.1-1.0）
   - **最大令牌数**：控制生成长度
   - **top_p**：核采样参数

3. **提示词模板**：
   - 查看和编辑各步骤的提示词
   - 创建自定义模板
   - 导入/导出模板

### 历史记录

**功能**：查看所有 AI 生成内容

**访问方式**：
1. 点击顶部工具栏的"历史记录"按钮
2. 查看按时间排序的生成记录
3. 筛选特定步骤或章节的历史
4. 重新使用历史生成内容

**实用价值**：
- 追溯创作思路变化
- 复用优秀的生成内容
- 分析 AI 生成模式

### 版本检查

**功能**：检查软件更新

**使用方法**：
1. 点击顶部版本号旁边的刷新按钮
2. 系统自动检查更新
3. 如有新版本，显示更新信息
4. 提供更新指引

### 数据管理

**功能**：管理应用数据和设置

**可用操作**：
1. **清空当前项目**：
   - 保留项目框架，清空所有内容
   - 从灵感阶段重新开始

2. **删除当前项目**：
   - 完全删除选中的项目
   - 不可恢复，请谨慎操作

3. **恢复出厂设置**：
   - 清除所有数据，包括 API 密钥
   - 应用将重启恢复初始状态

## 故障排除

### 常见问题解决

#### 1. 应用无法启动
**可能原因**：
- Node.js 版本不兼容
- 依赖包安装不完整
- 端口被占用

**解决方案**：
```bash
# 检查 Node.js 版本
node --version

# 重新安装依赖
npm install

# 清理缓存后重试
npm cache clean --force
```

#### 2. AI 功能不可用
**可能原因**：
- API 密钥未配置或失效
- 网络连接问题
- 模型配置错误

**解决方案**：
1. 检查模型设置中的 API 密钥
2. 测试网络连接
3. 重新配置模型参数
4. 尝试使用示例 API 密钥测试

#### 3. 数据保存失败
**可能原因**：
- 磁盘空间不足
- 文件权限问题
- 应用数据目录损坏

**解决方案**：
1. 检查磁盘可用空间
2. 以管理员身份运行应用
3. 清理临时文件
4. 手动备份后重置应用

#### 4. 性能问题
**可能原因**：
- 项目数据过大
- 同时打开过多功能
- 系统资源不足

**解决方案**：
1. 导出并清理旧项目
2. 关闭不必要的功能面板
3. 重启应用释放内存
4. 升级硬件配置

### 错误信息解读

#### "API 密钥无效"
- 检查密钥是否正确复制
- 确认密钥是否有使用权限
- 查看密钥是否过期

#### "网络连接失败"
- 检查网络连接状态
- 确认防火墙设置
- 尝试使用代理或 VPN

#### "生成内容过长"
- 减少输入内容长度
- 调整最大令牌数设置
- 分多次生成

#### "存储空间不足"
- 清理磁盘空间
- 导出并删除旧项目
- 移动数据到其他磁盘

- **提示词（Prompt）**：给 AI 的指令和上下文
